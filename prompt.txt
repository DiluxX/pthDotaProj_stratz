Разработай мобильное приложение на Python с использованием фреймворка Kivy (KivyMD)

для Android, состоящее из нескольких экранов (окон), связанных через ScreenManager.



Технические требования:

1. Архитектура экранов:

   - MainScreen (главное меню/список)

   - DetailScreen (детальная информация по выбранному элементу)

   - SettingsScreen (настройки)

   Переходы между экранами реализовать через ScreenManager с анимацией

   (SlideTransition).



2. Работа с API:

   - Использовать библиотеку requests (или aiohttp для асинхронных запросов,

     чтобы не блокировать UI-поток)

   - API-ключ хранить НЕ в коде, а через переменные окружения (python-dotenv)

     или локальное защищённое хранилище (KivyMD FileChooser + JSON/SQLite)

   - Все сетевые вызовы выполнять в отдельном потоке/через Clock.schedule_once,

     чтобы не блокировать основной UI-поток Kivy

   - Обработка ошибок: таймауты, отсутствие сети, невалидный ключ, лимиты API —

     показывать пользователю понятные сообщения (Snackbar/Dialog)



3. UI/UX:

   - Использовать компоненты KivyMD (MDTopAppBar, MDCard, MDList, MDTextField,

     MDRaisedButton) для единообразного Material Design

   - Адаптивная верстка под разные размеры экранов (используй dp/sp единицы)

   - Индикатор загрузки (MDSpinner) во время сетевых запросов



4. Структура проекта:

   - main.py — точка входа

   - screens/ — отдельные .py и .kv файлы для каждого экрана

   - services/api.py — модуль для работы с API

   - config.py — конфигурация (чтение API-ключа из .env)

   - buildozer.spec — конфигурация сборки под Android



5. Дополнительно:

   - requirements.txt со всеми зависимостями

   - Краткая инструкция по запуску и сборке APK через buildozer



Название API-сервиса и его документацию: [https://www.opendota.com].

Формат ответа API: [JSON/XML — уточнить].
